Java 8 Date Time Explained Series - Part 1

Jun 11, 2019

Reading time: 1 min

Tags: java java8 date-time
Categories: java dev

Background

Java has introduced improved date time APIs in Java 8 release by incorporating joda-time.

These new APIs under java.time package. java.time package contains APIs for dates, times, instants, time-zones and durations.

There are 4 categories in these APIs

  • Instants (a point in time since 1970)
  • Local date time (a date & time local to the consumer of the API)
  • Time zone date time (A date time in given time zone)
  • General purpose APIs like Clock, Duration, Period etc…

In this part-1 of date time series we will conver the Instant.

Instant

Instant is the point in the time since 1970. Its the unix like epoch equivalent in Java. Its time is always in UTC.

Instant is an immutable class and a value class.

import java.time.Instant;
public class Java8Instant {
    public static void main(String[] args) {
        System.out.println(Instant.now()); // prints current UTC date time
    }
}

Instant needs to hold a range longer than java long range so it will hold epoch seconds in long and nanosecond of the second in java int types.

        System.out.println(Instant.EPOCH); // prints 1970 epoch seconds
